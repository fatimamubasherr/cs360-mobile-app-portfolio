# cs360-mobile-app-development

Android mobile application demonstrating user-centered UI design, functional app development, and testing practices. Includes complete app code, UI design, and development artifacts.

Mobile App Development Portfolio

This repository contains my mobile application developed as part of CS 360. The project demonstrates my ability to design, develop, and test a functional Android application while applying user-centered design principles and mobile development best practices.
Reflection

App Requirements and User Needs

The goal of my app was to create a simple and functional mobile experience that allows users to efficiently interact with core features such as login and data management. The app was designed to address the need for a clean, user-friendly interface that simplifies basic tasks and ensures ease of use.

The primary user need was simplicity — users should be able to navigate the app without confusion, complete tasks quickly, and interact with clearly structured features.



UI Design and User-Centered Approach

To support user needs, the app included essential screens such as:

* Login screen
* Main interface (dashboard or inventory screen)
* Functional interaction elements (buttons, input fields)

My UI design focused on clarity, accessibility, and ease of navigation. I ensured that:

* Layouts were simple and not cluttered
* Buttons and inputs were clearly labeled
* Navigation was intuitive and consistent

These design choices were successful because they reduced user confusion and made the app easy to use, even for first-time users.



Coding Approach and Development Strategy

I approached coding by building the app step-by-step, starting with the UI layout and then implementing functionality using Java in Android Studio.

Some key strategies I used:

* Breaking the app into smaller components (activities, layouts)
* Writing clean and readable code for each feature
* Testing functionality incrementally as I built each part

This approach made debugging easier and ensured that issues were identified early. These strategies can be applied in future projects to improve efficiency and maintainability.



Testing and Functionality

To ensure my app was functional, I tested each feature individually, including:

* User input handling
* Button interactions
* Navigation between screens

Testing is important because it helps identify errors, ensures reliability, and improves overall user experience. Through testing, I was able to catch issues such as incorrect input handling and UI inconsistencies, which I then fixed.



Innovation and Problem-Solving

During development, I had to adapt when certain features did not work as expected. For example, handling user input and ensuring proper navigation required adjustments and troubleshooting.

I approached these challenges by researching solutions, testing different approaches, and refining my implementation until the app behaved correctly. This process improved my problem-solving skills and adaptability.



Strengths and Key Accomplishments

One area where I was particularly successful was in designing a clean and functional user interface. The app reflects my ability to apply user-centered design principles while maintaining simplicity and usability.

I also demonstrated strong understanding of:

* Android app structure
* Activity and layout management
* Input validation and user interaction



Summary

This project allowed me to combine design and development skills to create a working mobile application. It strengthened my understanding of user-centered design, coding practices, and testing strategies, and it provided a solid foundation for future mobile development projects.
